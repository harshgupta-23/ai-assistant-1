────────────────────────────────────────────────────────
4A. APP LAUNCH — NEVER USE EXE NAME ALONE
────────────────────────────────────────────────────────
  subprocess.Popen(["WINWORD.EXE"]) is WRONG.
  Windows PATH resolution can match wrong apps or open a browser.
  ALWAYS use full hardcoded paths with fallback:

    WORD:
      paths = [
        r"C:\\Program Files\\Microsoft Office\\root\\Office16\\WINWORD.EXE",
        r"C:\\Program Files (x86)\\Microsoft Office\\root\\Office16\\WINWORD.EXE",
        r"C:\\Program Files\\Microsoft Office\\Office16\\WINWORD.EXE",
      ]
      for p in paths:
          if os.path.exists(p):
              subprocess.Popen([p]); break
      else:
          subprocess.Popen(['powershell','-Command','Start-Process WINWORD.EXE'])

    EXCEL:      replace WINWORD.EXE → EXCEL.EXE
    POWERPOINT: replace WINWORD.EXE → POWERPNT.EXE
    CHROME:     r"C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe"
    NOTEPAD:    r"C:\\Windows\\System32\\notepad.exe"  (safe, always in PATH)
    VS CODE:    os.path.join(os.path.expanduser("~"),
                  r"AppData\\Local\\Programs\\Microsoft VS Code\\Code.exe")

  After launching, POLL for the window — never assume it is ready:
    for _ in range(20):           # up to 10 seconds
        wins = get_windows_by_process('WINWORD.EXE')
        if wins:
            break
        time.sleep(0.5)

────────────────────────────────────────────────────────
4B. WINDOW DETECTION — ALWAYS USE PROCESS-BASED MATCHING
────────────────────────────────────────────────────────
  gw.getWindowsWithTitle('Word') is WRONG.
  It matches ANY window containing 'Word' in its title, including
  Chrome tabs, File Explorer, and unrelated apps.

  ALWAYS detect windows by process name using psutil + Win32 PID:

    import psutil, ctypes

    def get_windows_by_process(process_name, title_filter=None):
        result = []
        for win in gw.getAllWindows():
            if not win.title.strip():
                continue
            try:
                pid = ctypes.c_ulong()
                ctypes.windll.user32.GetWindowThreadProcessId(
                    win._hWnd, ctypes.byref(pid)
                )
                proc = psutil.Process(pid.value)
                if proc.name().upper() == process_name.upper():
                    if title_filter is None or title_filter in win.title:
                        result.append(win)
            except Exception:
                continue
        return result

  Usage:
    word_wins   = get_windows_by_process('WINWORD.EXE')
    chrome_wins = get_windows_by_process('chrome.exe')
    excel_wins  = get_windows_by_process('EXCEL.EXE')
    vscode_wins = get_windows_by_process('Code.exe')

  EVERY script that interacts with a window must use this function.
  NEVER use gw.getWindowsWithTitle() for app detection.
  gw.getWindowsWithTitle() is ONLY acceptable for browser tab title checks.

────────────────────────────────────────────────────────
4C. WINDOW FOCUS — ALWAYS USE WIN32 WITH VERIFICATION
────────────────────────────────────────────────────────
  pygetwindow.activate() silently fails on most Windows setups.
  NEVER use it as the primary focus method.

  ALWAYS use this verified force-focus function:

    def force_focus_verified(win, retries=3):
        for attempt in range(retries):
            try:
                ctypes.windll.user32.ShowWindow(win._hWnd, 9)  # SW_RESTORE
                ctypes.windll.user32.SetForegroundWindow(win._hWnd)
                time.sleep(0.5)
                fg = ctypes.windll.user32.GetForegroundWindow()
                if fg == win._hWnd:
                    print(f"[OK] Focus verified on attempt {attempt+1}.")
                    return True
                print(f"[WARN] Focus not confirmed, retrying ({attempt+1}/{retries})...")
            except Exception as e:
                print(f"[ERROR] Focus attempt {attempt+1} failed: {e}")
        print("[ERROR] Could not verify window focus after all retries.")
        return False

  Only fall back to win.activate() if force_focus_verified() returns False.
  ALWAYS add time.sleep(0.5) after any focus attempt.

────────────────────────────────────────────────────────

4E. CLIPBOARD OVER TYPEWRITE — ALWAYS
────────────────────────────────────────────────────────

4G. TIMING RULES
────────────────────────────────────────────────────────
  NEVER use a single large sleep as substitute for polling.
  Use polling loops for app/page readiness. Use sleeps only for
  fixed UI transitions:

  - After app launch:               poll every 0.5s, max 10s (20 iterations)
  - After Ctrl+T (new tab):         time.sleep(0.6)
  - After Ctrl+L (address bar):     time.sleep(0.4)
  - After URL navigation + Enter:   time.sleep(2.5)
  - After Win32 force focus:        time.sleep(0.5)
  - After clipboard paste:          time.sleep(0.2)
  - After any keyboard shortcut:    time.sleep(0.2)
  - After Word/Excel fully loads:   time.sleep(2.5)
  - After dialog confirmation:      time.sleep(0.5)
